# Support

Thanks for your interest in BuffaloHerd OSS.

This organization contains a mix of open-source software, research code, technical experiments, instructional materials, templates, and supporting documentation.
Because these repositories may differ in scope and maturity, support expectations may vary from one repository to another.

Please read the repository README first.
In many cases, it will answer common questions about project status, intended use, development location, and contribution workflow.

## Getting Help

For most repositories, GitHub Issues is the best place for:

- bug reports
- feature suggestions
- documentation corrections
- questions about repository contents or behavior

If a repository’s README or other documentation points to a different upstream location, issue tracker, or support process, please follow the repository-specific guidance instead.

## Before Opening an Issue

Before requesting support, please review the available documentation for the repository, including the README and any development or contribution notes.

When asking for help, it is useful to include:

- a clear description of the problem or question
- steps to reproduce the issue, if applicable
- relevant environment details
- logs, screenshots, or example input/output where helpful

Clear, well-scoped reports make it much easier to understand and respond productively.

## Scope of Support

BuffaloHerd OSS is an open and exploratory project space.
Support is provided on a best-effort basis.

Some repositories may be actively maintained, while others may be experimental, instructional, incomplete, or primarily provided for reference.
Not every repository should be assumed to offer the same level of responsiveness, stability, or long-term support.

## Security Issues

Please do not report security vulnerabilities in public issues.

If you need to report a security-sensitive issue, contact:

**buffalo.herd.oss@gmail.com**

Please include enough detail to explain the concern clearly, and identify the affected repository if applicable.

## Repository-Specific Guidance

If a repository includes its own `SUPPORT.md`, README guidance, or other support instructions, that repository-specific guidance takes precedence over this organization-wide file.
